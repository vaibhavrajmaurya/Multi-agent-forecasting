1. Executive Summary

This project builds a multi-agent online-learning system that forecasts the next day's return of the S&P 500 (via the SPY ETF). Four different forecasting models ("agents") each make a daily prediction, and a game-theoretic algorithm called Hedge blends them into a single forecast, automatically trusting whichever agents have been most accurate recently.

AspectWhat we did
DataSPY + VIX daily prices, 2010–2024, via Yahoo Finance
TargetNext-day log return (a stationary transform of price)
AgentsTrend (linear+seasonality), Momentum (XGBoost), Volatility (XGBoost), Sequence (LSTM)
AggregatorHedge (Freund–Schapire 1997) + Fixed-Share (Herbster–Warmuth 1998)
Validation48-fold walk-forward backtest (2,992 out-of-sample days)
OutputAn interactive Streamlit dashboard + this report

The headline result (honest version)

The adaptive Hedge ensemble achieves a Sharpe of 0.38, ~58% higher than naive equal-weighting (0.24) — proving the online learning adds real value. The best single agent (LSTM) reaches 0.73 Sharpe, competitive with passive Buy & Hold (0.79). We do not beat passive Buy & Hold on raw return, because beating daily index returns by timing is near-impossible — and saying so honestly is part of the project's credibility.

2. The Problem: What Are We Actually Trying to Do?

Every trading day, we want to answer one question: "Will the S&P 500 go up or down tomorrow, and by roughly how much?" If we can answer even slightly better than a coin flip, we can build a strategy: go long (bet up) when we predict up, go short (bet down) when we predict down.

This is a time-series forecasting problem — we predict the future of a sequence (daily returns) from its past. It is famously hard because financial markets are close to efficient: any easily-found pattern gets traded away, so daily returns are mostly noise. Our goal is therefore modest and scientific: extract whatever weak signal exists, combine multiple models intelligently, and measure honestly whether the combination helps.

💡 Why "multi-agent"? Different models capture different market behaviors — long-run drift, short-term momentum, volatility regimes, non-linear patterns. Rather than betting on one, we run all four and let an algorithm decide, day by day, which to trust. That decision-maker is the Hedge aggregator.

3. Glossary — Every Term Used

Log return
The natural log of the ratio of consecutive prices: ln(P_t / P_{t-1}). Used instead of raw price because prices trend upward forever (non-stationary) while returns hover around zero with stable spread (stationary) — which is what ML models need. Log returns are also additive over time (a 5-day return is the sum of 5 daily log returns).
Stationary / Non-stationary
A series is stationary if its statistical properties (mean, variance) don't drift over time. SPY's price went from \$115 to \$480 (non-stationary); its log returns stay ~0 mean, ~1% daily std the whole time (stationary). Models trained on non-stationary data fail on unseen value ranges.
Lookahead bias (a.k.a. data leakage)
Accidentally letting the model use information from the future. The #1 way backtests lie. Fixed by always shifting features back by at least one day (.shift(1)) so a prediction for day t only uses data available at the end of day t−1.
Walk-forward validation
The correct way to test a time-series model: train on the past, test on the immediate future, then roll the window forward and repeat. Never shuffle the data (that would leak the future into the past).
Directional accuracy
The fraction of days the model got the sign (up/down) right. 50% = coin flip. This is the most actionable metric — being right about direction is what makes money.
Sharpe ratio
Risk-adjusted return: mean(returns) / std(returns) × √252. Higher is better. Roughly: <0.5 weak, 0.5–1.0 decent, >2 excellent (and usually suspicious on daily data). The √252 annualizes daily numbers (≈252 trading days/year).
Maximum drawdown
The worst peak-to-trough drop in portfolio value, as a fraction. Measures downside pain.
Information ratio
Like Sharpe but measured against a benchmark: mean(strategy − benchmark) / std(strategy − benchmark) × √252. Tells you if you add value relative to just holding the index.
RSI (Relative Strength Index)
A momentum oscillator (0–100) comparing recent gains to recent losses. >70 = "overbought", <30 = "oversold". A mean-reversion signal.
MACD (Moving Average Convergence Divergence)
The gap between a fast (12-day) and slow (26-day) exponential moving average of price, plus a 9-day "signal line". A trend/momentum indicator.
Bollinger Band width
How wide a ±2-standard-deviation envelope around a 20-day average is. Narrow = calm market; a "squeeze" often precedes a big move.
VIX
The market's "fear gauge" — the options market's expectation of near-term volatility. Spikes during crashes. We feed its level and 5-day change as features.
EWM vs SMA
Exponentially Weighted Mean weights recent data more (responsive); Simple Moving Average weights all data in the window equally (stable). RSI/MACD use EWM; Bollinger uses SMA.
Lag feature
A past value of the series used as an input. lag_1 = yesterday's return, lag_5 = the return 5 days ago. The autoregressive backbone of forecasting.
Online learning
Learning that updates after every single new data point (here: every trading day), rather than training once on a fixed dataset. The Hedge algorithm is an online learner.
Hedge algorithm / Multiplicative Weights
An online algorithm that keeps a weight on each expert (agent), and after each round multiplies each weight by exp(−η × loss) — shrinking the weights of experts that did badly. η ("eta") is the learning rate.
Regret / No-regret
"Regret" = how much worse the ensemble did than the best single expert in hindsight. Hedge has a no-regret guarantee: regret grows only like √(T·log N), so per-day regret → 0 as time T grows. Translation: the ensemble is guaranteed to do nearly as well as the best agent, without knowing in advance which agent that is.
Fixed-Share
An add-on to Hedge for non-stationary problems: after each update, mix a little uniform weight back into every agent so none ever dies. Lets the ensemble recover an agent when the market regime changes.
XGBoost
"Extreme Gradient Boosting" — a powerful tree-based ML model for tabular data. Builds many small decision trees, each correcting the previous one's errors.
LSTM (Long Short-Term Memory)
A type of recurrent neural network designed to read sequences in order and remember long-range patterns. Used here to read the last 30 days of returns.

4. System Architecture (The Pipeline)

Data flows through six stages. Each stage is one Python module.

Yahoo Finance (SPY + VIX, 2010–2024)
        │   data/fetch.py
        ▼
Feature Engineering  ── pipeline/features.py
   log returns · RSI · MACD · Bollinger · lags · rolling stats · VIX
        │
        ▼
Four Agents  ── pipeline/agents.py
   TrendAgent · MomentumAgent · VolatilityAgent · SequenceAgent(LSTM)
        │   (each makes an independent daily prediction)
        ▼
Hedge + Fixed-Share Aggregator  ── pipeline/aggregator.py
   blends the 4 predictions into ONE, reweighting daily by accuracy
        │
        ▼
Walk-Forward Backtest  ── pipeline/backtest.py + pipeline/metrics.py
   48 folds · Sharpe, drawdown, directional accuracy, information ratio
        │
        ▼
Streamlit Dashboard  ── app.py

The key idea: features and agents are upstream; the Hedge aggregator is the brain that decides how to combine them; the backtest is the honest referee; the dashboard is the presentation layer.

5. Code Walkthrough 1 — Data Ingestion (data/fetch.py)

This module downloads raw price data and converts it into the base series everything else uses.

Downloading SPY and computing log returns

def fetch_spy():
    df = yf.download("SPY", start=START, end=END, auto_adjust=True)
    df = df[["Close"]].copy()
    df["log_return"] = np.log(df["Close"] / df["Close"].shift(1))
    df.dropna(inplace=True)
    return df
  • auto_adjust=True — adjusts prices for dividends/splits so returns are accurate.
  • log(Close / Close.shift(1)) — today's close ÷ yesterday's close, then log. This is the log return, our prediction target.
  • .shift(1) aligns yesterday's price with today's row. The first row becomes NaN (no prior day) and is dropped.

Downloading VIX (the fear gauge)

vix["vix_change_5d"] = vix["vix_close"].pct_change(5)

We keep VIX's level and its 5-day percentage change (is fear rising or falling?).

Caching: load_or_fetch()

Downloads once, saves to CSV, and on later runs reads the CSV instead of hitting Yahoo Finance again — faster and avoids rate limits. The SPY and VIX tables are joined on date (inner join, so only days both exist are kept).

6. Code Walkthrough 2 — Feature Engineering (pipeline/features.py)

"Features" are the inputs the models learn from. This module turns the raw price/return series into a clean table of predictive signals — all carefully shifted to prevent lookahead bias.

RSI

def compute_rsi(series, period=14):
    delta = series.diff()
    gain = delta.clip(lower=0)        # keep only up moves
    loss = -delta.clip(upper=0)       # keep only down moves (as positive)
    avg_gain = gain.ewm(com=period-1).mean()
    avg_loss = loss.ewm(com=period-1).mean()
    rs = avg_gain / avg_loss
    return 100 - (100 / (1 + rs))

Splits each day's move into gains and losses, takes an exponentially-weighted average of each, and forms the ratio. High RSI = lots of recent gains (overbought); low = oversold.

MACD

ema_fast = series.ewm(span=12).mean()
ema_slow = series.ewm(span=26).mean()
macd_line = ema_fast - ema_slow
signal_line = macd_line.ewm(span=9).mean()

Fast minus slow EMA measures momentum; the 9-day "signal line" smooths it. We use the signal line as the feature (less noisy).

Bollinger Band width & lag/rolling features

bb_width = (upper - lower) / mid      # width of ±2σ envelope, normalized
lag_k    = series.shift(k)             # return k days ago
rolling  = series.shift(1).rolling(w).mean()/std()  # window ENDS at t-1
⚠ The single most important line in the project 
close = df["Close"].shift(1)   # yesterday's close — all features built from here
Inside build_feature_matrix(), every technical indicator is computed from yesterday's close, not today's. Without this one shift, the model would secretly see the current day's price — producing a backtest that looks perfect and collapses in real trading. This is how lookahead bias is prevented at the source.

build_feature_matrix() assembles RSI, MACD signal, Bollinger width, 5 lag features, 4 rolling stats, and 2 VIX features into one table, with log_return as the last column (the target). Rows with any NaN (from the warm-up windows) are dropped.

7. Code Walkthrough 3 — The Four Agents (pipeline/agents.py)

The shared contract: BaseAgent

Every agent is a Python class with the same two methods, so the aggregator can treat them interchangeably (this is the "Abstract Base Class" pattern):

class BaseAgent(ABC):
    def fit(self, train_df): ...     # learn from historical rows
    def predict(self, test_df): ...  # output one prediction per row

Each agent receives the full feature table and internally picks the columns it cares about.

Agent 1 — TrendAgent (classical statistics)

ModelLinear Regression on a DeterministicProcess
InputsOnly the calendar date (a linear time trend + annual Fourier seasonality)
CapturesThe market's long-run upward drift and calendar/seasonal effects
 
fourier = CalendarFourier(freq="YE", order=4)        # annual seasonal waves
dp = DeterministicProcess(index, constant=True, order=1,  # linear trend
                          additional_terms=[fourier])
model.fit(dp.in_sample(), train["log_return"])
preds = model.predict(dp.out_of_sample(steps=n, forecast_index=test.index))

This is literally time-series decomposition (trend + seasonality) from the statsmodels.tsa ("time series analysis") library. It uses no price features at all — just the passage of time — so it embodies the theory "stocks drift up over time."

Agent 2 — MomentumAgent (gradient-boosted trees)

ModelXGBRegressor (200 trees, depth 4, lr 0.05)
Inputslag_1…lag_10, rolling means/std, RSI, MACD signal
CapturesShort-term momentum / continuation patterns
 
self._model = XGBRegressor(n_estimators=200, learning_rate=0.05, max_depth=4, ...)
self._model.fit(X[momentum_cols], y)

This is the "reframe time-series as supervised regression" approach: feed recent returns and indicators as columns, predict the next return.

Agent 3 — VolatilityAgent (gradient-boosted trees, different lens)

ModelXGBRegressor (same settings)
InputsBollinger width, rolling std (5d, 21d), VIX level, VIX 5-day change
CapturesVolatility regime — behaves differently in calm vs stressed markets

Same algorithm as Momentum, but fed only volatility/fear features. It embodies "behavior depends on the volatility regime."

Agent 4 — SequenceAgent (LSTM deep learning)

Model2-layer LSTM, hidden size 64, dropout 0.2 (PyTorch)
InputsThe raw sequence of the last 30 daily returns
CapturesNon-linear temporal patterns too complex for the other models
 
# turn a 1-D return series into overlapping 30-day windows
for i in range(window, len(series)):
    X.append(series[i-window : i]);  y.append(series[i])

# network: LSTM reads the window, a linear head outputs one number
out, _ = self.lstm(x)
return self.head(out[:, -1, :])      # use the LAST timestep's hidden state

Key training details: returns are standardized (scaler fit on training data only); training uses Adam + early stopping (stop when validation loss stops improving); and — after our fixes — a fixed random seed makes it reproducible. This is the deep-learning leg of the project and, after the fixes, the single best-performing agent (0.73 Sharpe).

💡 Four lenses, one decision TrendAgent sees only the calendar. MomentumAgent sees recent returns & indicators. VolatilityAgent sees fear & volatility. SequenceAgent sees raw 30-day patterns. None sees the others' features. The Hedge aggregator's whole job is to discover, live, which lens is working now.

8. Code Walkthrough 4 — The Hedge Aggregator (pipeline/aggregator.py)

This is the heart of the project — the part that makes it "multi-agent." It keeps a weight on each agent (how much to trust it) and updates those weights every day.

Making one combined prediction

def aggregate(self, predictions):
    return np.dot(self.weights, predictions)   # weighted average of the 4 agents

The daily weight update (final version)

def update(self, predictions, actual):
    losses = (predictions - actual) ** 2             # squared error per agent
    norm_losses = losses / (losses.mean() + 1e-12)   # (1) scale-invariant
    self.weights *= np.exp(-self.eta * norm_losses)   # (2) Hedge penalty
    self.weights /= self.weights.sum()
    self.weights = (1-self.alpha)*self.weights + self.alpha/N  # (3) Fixed-Share
    self.weights /= self.weights.sum()
  1. Normalize losses — raw squared log-return losses are ~0.0001; dividing by their mean rescales them so the learning rate η actually has an effect (without this the weights never move).
  2. Hedge penalty — each weight is multiplied by exp(−η × loss): agents that erred more shrink more. Then renormalize so weights sum to 1.
  3. Fixed-Share — mix a small uniform amount (α/N) back into every weight so no agent ever dies; this keeps the ensemble able to adapt when the regime changes.

The EqualWeightAggregator is a baseline with the same interface but a no-op update — it always averages the four agents equally. Comparing Hedge against it proves whether the adaptive weighting actually helps. (It does: 0.38 vs 0.24 Sharpe.)

The regret guarantee, in plain English Hedge's theory promises that over T days, the ensemble's total loss exceeds the best agent's total loss by at most ~√(T·log N). Divided by T, that gap shrinks to zero. So you are mathematically guaranteed to do almost as well as the best agent — without knowing in advance which one it is. That robustness, not maximum return, is the point.

9. Code Walkthrough 5 — Metrics (pipeline/metrics.py)

Turns a prediction series into the financial scorecard.

strategy_returns = np.sign(pred) * actual    # long if pred>0, short if pred<0
equity = 10000 * np.exp(np.cumsum(strategy_returns))  # $10k compounded
  • Sharpe = mean/std of strategy returns × √252.
  • Max drawdown = worst peak-to-trough fall of the equity curve.
  • Directional accuracy = fraction of days sign(pred) == sign(actual).
  • Information ratio = risk-adjusted excess return vs Buy & Hold.
  • Conviction option (added, default off) — trade only the most-confident days; tested and found not to help (see Corrections).

10. Code Walkthrough 6 — Walk-Forward Backtest (pipeline/backtest.py)

This is the honest referee. It never lets a model see the future.

train_end = start + 3 years           # initial training window
while more data:
    train = data[: train_end]            # everything up to now
    test  = data[train_end : +3 months]  # the next quarter (unseen)
    for agent in agents: agent.fit(train)
    for day in test:                       # step through the quarter day by day
        ensemble = aggregator.aggregate(agent_preds)
        aggregator.update(agent_preds, actual)   # learn from the realized outcome
    train_end += 3 months                 # roll forward (expanding window)

Each fold trains on all data up to a point, tests on the next 3 months, then expands. Over 2010–2024 this yields 48 folds and 2,992 out-of-sample test days. Crucially, the Hedge weights update within each test quarter, day by day — true online learning. The final report compares every agent, the Hedge ensemble, equal-weight, and Buy & Hold.

The three corrections that mattered most

#5 — The Hedge weights never moved (η 10,000× too small)

With loss ≈ 0.0001 and η = 0.1, the update multiplier was exp(−0.00001) ≈ 0.99999 — essentially 1. The weights stayed at 25% each for 12 years, and Hedge gave identical results to dumb averaging. Fix: normalize losses by their per-step mean so η has real effect. Lesson: the Hedge regret bound assumes losses in [0,1]; on a different scale the algorithm silently degenerates.

#6 — The LSTM faked ~half its predictions

The LSTM needs 30 days of history, so the original code filled the first 30 days of every fold with zeros — ~1,440 of 2,992 days. This tanked directional accuracy (a zero never matches a real sign) and faked a good Sharpe (zeros deflate volatility). Fix: carry the trailing 30 training days into prediction so every test day gets a real forecast (no leakage). Lesson: metric artifacts are sneaky — a positive Sharpe with 28% accuracy is a contradiction worth chasing.

#9 — Hedge collapsed onto one agent

After fixing η, the weights swung too hard: all weight piled onto SequenceAgent in the first weeks and never moved again (a dead agent can't recover under pure multiplicative weights). Fix: Fixed-Share mixes uniform weight back each step, so the ensemble stays adaptive and the weight chart "breathes". Lesson: vanilla Hedge is for stationary problems; markets are not.

The two honest negative results

Not every idea worked — and we kept the failures visible:

  • P&L-aligned loss (#10/10b): weighting agents by realized profit instead of squared error sounded right, but daily direction is so noisy that it just chased luck. Reverted to MSE.
  • Conviction threshold (#12): trading only "high-confidence" days made things worse (−0.13 Sharpe) — the models' prediction magnitude doesn't track their accuracy.

Documenting these is a credibility signal: it shows the results weren't cherry-picked.

13. Final Results & Honest Interpretation

48-fold walk-forward, 2,992 out-of-sample days, reproducible (fixed seed + cuDNN determinism). Strategy = go long/short daily by predicted direction.

What this honestly means

  • The algorithm works: the adaptive Hedge ensemble beats naive equal-weighting by ~58% on Sharpe (0.38 vs 0.24). Online learning adds real value over dumb averaging.
  • The weak agents are coin flips: Momentum and Volatility sit at ~50% directional accuracy — exactly what weak-form market efficiency predicts for technical indicators on liquid daily data. The trend and LSTM agents carry the signal.
  • We do not beat passive Buy & Hold ($21k vs $48k), and that is the correct result. Buy & Hold captures the full 12-year equity premium; an active long/short model that goes short ~half the time fights that uptrend and bleeds. Beating daily index returns by timing is near-impossible without leverage or lookahead bias.
  • The best single agent beat the ensemble in hindsight (LSTM $43k vs ensemble $21k). The ensemble's value isn't max return — it's robustness: it tracks the best agent without knowing in advance which one it'll be, and it beats naive averaging reliably.
📊 Why "predict COVID" is the wrong expectation No model trained on price history can foresee an exogenous shock like COVID — the cause (a pandemic) isn't in the data, and any predictable crash would already be priced in. The model can only react, and with no-lookahead rules it reacts a day late — fatal when the move is a one-day jump followed by a V-shaped recovery that whipsaws reactive strategies. This is why passive Buy & Hold wins through COVID, and it's a feature of an honest backtest.

Limitations (state these proactively)

  • No transaction costs — daily long/short implies high turnover; real spreads would erode the active strategies more than Buy & Hold.
  • Unit position sizing — always ±1, no confidence-based sizing or risk limits.
  • Single asset — SPY only; results may differ on other markets.
  • LSTM on daily returns is inherently noisy; one strong seed is not proof of a durable edge.
